C++的输入输出系统基于流的概念。流是数据的序列，可以从源（输入流）读取或写入到目标（输出流）。C++提供了强大的I/O库来处理各种类型的输入输出操作。

## 基本输出

### 使用 `cout` 进行基本输出

`std::cout` 是 C++ 标准库提供的面向对象的输出流，用于向控制台打印信息。我们使用**插入运算符 `<<`** 将数据写入流中。

```cpp
#include <iostream>
using namespace std; 
// 事先声明: 为了简化代码，在小型的项目（比如做算法题），我们通常使用 using namespace std;
// 这样就可以直接使用 std 命名空间中的名字（如 cout, cin, string, endl），
// 而无需每次都在前面加上 std:: 前缀，从而减少代码量，提高编写速度。
// 但在大型或团队项目中，为了避免命名冲突，通常不推荐在头文件或全局作用域使用。
int main() {
    // 基本输出
    cout << "Hello, World!" << endl; // endl 会换行并刷新缓冲区

    // 输出变量
    int age = 25;
    cout << "Age: " << age << endl;

    // 输出多个值
    cout << "Name: " << "John" << ", Age: " << age << endl;

    // 不使用 endl（不换行）
    cout << "This is on the same line ";
    cout << "as this." << endl;

    // 使用 '\n' 进行换行（比 endl 效率高，因为它不强制刷新缓冲区）
    cout << "洋服剑舞，所谓伊人醉山花" << '\n'; // 注意：'\n' 后需要分号

    return 0;
}
```

-----

### 使用 `printf` 进行基本输出

`printf` 是 C 语言风格的输出函数，在 C++ 中也可以使用。它通过**格式控制字符串**来精确控制输出的格式，你需要包含 `<cstdio>` 头文件来使用它。

```cpp
#include <cstdio> // 包含 cstdio 库，才能使用 printf

int main() {
    int a = 1, b = 2;
    double pi = 3.14159;
    char initial = 'M';
    const char* greeting = "你好世界"; // C 风格字符串，需要以 '\0' 结尾

    // 1. 输出纯文本
    // 直接在双引号内输入文本，结尾记得加 '\n' 换行，否则会连续输出
    printf("昨夜闲潭梦落花，可怜春半不还家。\n");

    // 2. 输出变量的值 (使用格式占位符)
    // %d 用于输出十进制整数
    printf("变量 a 的值为: %d\n", a);
    printf("变量 b 的值为: %d\n", b);

    // 3. 输出常量的值 (使用格式占位符)
    // 即使是常量，也可以用占位符来明确其类型
    printf("常量数字 1 的输出: %d\n", 1);

    // 4. 输出不同类型的变量
    // %f 用于输出浮点数
    printf("圆周率的值约为: %f\n", pi);
    // %c 用于输出单个字符
    printf("首字母是: %c\n", initial);
    // %s 用于输出 C 风格字符串 (char 数组或 const char*)
    printf("一句问候: %s\n", greeting);

    // 5. 输出多个值
    // 可以在一个 printf 中使用多个占位符，按顺序传入对应变量
    printf("a = %d, b = %d, pi = %f\n", a, b, pi);

    // 6. 简单的格式化控制
    // %.2f 表示保留两位小数
    printf("保留两位小数的圆周率: %.2f\n", pi);
    // %5d 表示输出一个至少占 5 个字符宽度的整数，不足则左侧填充空格（默认右对齐）
    printf("宽度为5的整数: %5d\n", a);

    return 0;
}
```

-----

**补充说明：`cout` 与 `printf` 的性能考量**

你或许会好奇，既然 C++ 提供了方便易用的 `cout`，为何我们有时还会用到 `printf`？这主要是因为两者的**性能差异**。

在**处理大量输入/输出数据时，`cout` 的处理速度通常会慢于 `printf`**。这种差异在一些对运行时间有严格限制的场景（如**算法竞赛**）中尤为明显。如果你的程序因为时间超限（**TLE**）而未能通过测试，输入/输出操作很可能就是瓶颈所在。对于**输入操作的 `cin`** 而言，也存在类似的速度考量。

**简而言之：**

  * **`cout`/`cin`**：更安全、更方便，适合日常开发。
  * **`printf`/`scanf`**：在特定高性能场景下可能更快，但需要更严格的格式匹配和手动缓冲区管理。

## 基本输入

### 使用 `cin` 进行基本输入

`std::cin` 是 C++ 标准库中用于从标准输入流（通常是键盘）读取数据的对象。我们使用 **提取运算符 `>>`** 从输入流中获取数据。

```cpp
#include <iostream> // 使用 cin, cout, endl 需要包含此头文件
#include <string>   // 使用 std::string 需要包含此头文件

int main() {
    int age;
    std::string name; // 推荐使用 std::string 处理字符串

    // 输入整数
    std::cout << "Enter your age: ";
    std::cin >> age; // std::cin 会读取输入的整数并存储到 age 变量中

    // 输入字符串
    std::cout << "Enter your name: ";
    // 注意：std::cin >> name 只能读取不包含空格的单个单词
    std::cin >> name;

    std::cout << "Hello, " << name << "! You are " << age << " years old." << std::endl;

    return 0;
}
```

### 使用 `scanf` 进行基本输入

`scanf` 是 C 语言风格的输入函数，在 C++ 中也可以使用。它通过**格式控制字符串**来精确控制输入数据的读取方式，并且通常需要传入变量的**地址**（使用 `&` 运算符）。你需要包含 `<cstdio>` 头文件来使用它。

```cpp
#include <cstdio> // 包含 cstdio 库，才能使用 scanf 和 printf

int main() {
    int age_c;
    // 使用 char 数组来存储 C 风格字符串。
    // 注意：要预留足够的空间，包括字符串结束符 '\0'。
    char name_c[50]; 

    // 输入整数
    printf("Enter your age (C style): ");
    // %d 用于读取十进制整数。
    // 注意：age_c 前面需要加 & 来获取其内存地址，scanf 会将值写入到这个地址。
    scanf("%d", &age_c); 

    // 输入字符串
    printf("Enter your name (C style, single word): ");
    // %s 用于读取字符串。scanf 遇到空格、制表符或换行符会停止读取。
    // char 数组名本身就代表其起始地址，所以 name_c 前面不需要加 &。
    scanf("%s", name_c); 
    
    // 打印结果，使用 printf 输出
    printf("Hello, %s! You are %d years old (C style).\n", name_c, age_c);

    // --- 注意 scanf 和缓冲区的常见问题 ---
    // 和 cin 类似，scanf 读取数字或单词后，用户按下的回车符 '\n' 会留在输入缓冲区中。
    // 如果后续你需要读取一整行（例如使用 gets 或 fgets），这个 '\n' 可能会导致问题。
    // 简单的处理方法是清除缓冲区中剩余的字符，例如使用 getchar() 循环：
    // while (getchar() != '\n' && getchar() != EOF); 
    // 或者在下一个 scanf 格式字符串的占位符前添加一个空格，比如：
    // char ch;
    // scanf(" %c", &ch); // 注意 %c 前的空格会跳过所有空白字符，包括换行符。

    return 0;
}
```

## 格式化输出

C++ 的 `std::cout` 提供了强大的格式化功能，让你能精确控制数据在控制台上的显示方式。要使用这些功能，你需要包含 **`<iomanip>`** 头文件，它提供了各种**流操作符（manipulators）**。

### 使用 `cout` 的格式化功能

```cpp
#include <iostream>  // 用于 cout, endl
#include <iomanip>   // 用于 setprecision, setw, setfill 等格式化操作符
using namespace std; 

int main() {
    double pi = 3.14159265359;
    
    // 设置浮点数精度
    cout << "Default precision: " << pi << endl; // 默认精度可能因系统而异

    // std::fixed: 禁用科学计数法，以固定小数位数显示。
    // std::setprecision(n): 对于 fixed 模式，n 表示小数点后的位数；对于默认模式，n 表示总的有效数字位数。
    cout << "Fixed precision (2): " << fixed << setprecision(2) << pi << endl;
    
    // std::scientific: 强制以科学计数法显示。
    cout << "Scientific notation: " << scientific << pi << endl;
    
    // 恢复默认浮点数显示模式，这很重要，否则后续浮点数仍会受之前设置影响。
    cout << defaultfloat; 
    cout << "Back to default float: " << pi << endl;

    // 设置字段宽度和填充字符
    int number = 42;
    cout << "Default: " << number << endl;
    
    // std::setw(n): 设置下一个输出项的字段宽度为 n。只对紧随其后的一个输出项有效。
    // 默认情况下，内容会右对齐。
    cout << "Width 10 (right-aligned): " << setw(10) << number << endl;
    
    // std::setfill(char): 设置填充字符。此设置会一直有效，直到被更改。
    cout << "Width 10, fill with 0: " << setw(10) << setfill('0') << number << endl;
    
    // 恢复默认填充字符为空格。
    cout << setfill(' '); 
    cout << "Width 10, fill with space (after reset): " << setw(10) << number << endl;

    // 整数基数输出 (默认是十进制)
    cout << "Decimal: " << number << endl;
    
    // std::hex: 以十六进制输出。
    // 如果想显示前缀 0x，可以使用 std::showbase。
    cout << "Hexadecimal: " << hex << number << endl;
    
    // std::oct: 以八进制输出。
    // 如果想显示前缀 0，也可以使用 std::showbase。
    cout << "Octal: " << oct << number << endl;
    
    // std::dec: 恢复为十进制输出。这很重要，因为基数设置会一直有效。
    cout << "Back to decimal: " << dec << number << endl;

    // 其他常用格式化选项示例：
    bool boolean_val = true;
    cout << "Boolean (numeric): " << boolean_val << endl;
    // std::boolalpha: 将布尔值输出为 "true" 或 "false"。
    cout << "Boolean (alpha): " << boolalpha << boolean_val << endl;
    // std::noboolalpha: 恢复布尔值输出为 1 或 0。
    cout << noboolalpha; 
    cout << "Boolean (alpha reset): " << boolean_val << endl;

    return 0;
}
```

## 错误处理

编写健壮的程序意味着要能处理各种意外情况，包括用户输入错误或文件操作失败。C++ 的 `iostream` 对象内置了**状态标志**来指示 I/O 操作是否成功。理解并利用这些状态标志是进行错误处理的关键。

### 处理输入输出错误

```cpp
#include <iostream> // 用于 cout, cin, endl
#include <limits>   // 用于 std::numeric_limits
#include <string>   // 用于 std::string

using namespace std; 

int main() {
    int number;

    cout << "Enter a number: ";

    // 检查输入是否成功
    // 当 cin >> number 成功读取到有效数据时，表达式的结果为 true。
    // 如果用户输入了非数字字符，cin 将进入失败状态。
    if (cin >> number) {
        cout << "You entered: " << number << endl;
    } else {
        cout << "Invalid input! Please enter a valid number." << endl;

        // 1. 清除错误状态：
        // cin.clear() 会清除流的内部错误标志（如 failbit, badbit），
        // 使流从错误状态中恢复，可以进行后续操作。
        cin.clear();

        // 2. 清除输入缓冲区中无效的字符：
        // 当输入失败时，那些导致失败的无效字符（以及随后的换行符）会留在缓冲区中。
        // 如果不清除它们，后续的输入操作会立即再次读取这些无效内容，导致无限循环的错误。
        // numeric_limits<streamsize>::max() 表示忽略最大数量的字符，
        // '\n' 表示遇到换行符时停止忽略（并丢弃该换行符）。
        cin.ignore(numeric_limits<streamsize>::max(), '\n');

        // 再次尝试输入
        cout << "Please try again, enter a number: ";
        if (cin >> number) {
            cout << "You finally entered: " << number << endl;
        } else {
            cout << "Still invalid input. Exiting program." << endl;
        }
    }

    // 文件操作的错误处理也类似，可以通过检查文件流对象的状态来判断操作是否成功。
    /*
    ifstream inFile("nonexistent.txt");
    if (!inFile.is_open()) { // 检查文件是否成功打开
        cerr << "Error: Could not open file." << endl;
    }
    // 或者更简洁地检查流对象本身（在非错误状态时为 true）
    // if (inFile) {
    //     // 文件操作
    // } else {
    //     // 错误处理
    // }
    */

    return 0;
}
```

## 流操作符

C++ 的 `iostream` 库提供了多种**流操作符（manipulators）**，它们允许你控制输入输出流的行为。这些操作符可以直接插入到流中，改变流的状态或执行特定的操作。

### C++ 提供的流操作符示例

```cpp
#include <iostream> // 包含 iostream 库
#include <string>   // 包含 string 库，用于 string 类型

using namespace std; 

int main() {
    // endl: 换行并刷新缓冲区
    // 它是 cout << '\n' << flush; 的简写。
    // 频繁使用可能影响性能，在不需要立即刷新时，推荐使用 '\n'。
    cout << "Line 1" << endl;

    // flush: 刷新缓冲区
    // 强制将缓冲区中的内容写入输出设备，但不插入换行符。
    cout << "Line 2" << flush;
    cout << " This is on the same line after flush." << endl;

    // ends: 添加空字符
    // 在输出流中插入一个空字符 '\0'。常用于写入文件时作为字符串结束符，但很少用于控制台输出。
    cout << "Line 3" << ends;
    cout << " (after ends)" << endl;

    // ws: 跳过空白字符 (用于输入流)
    // 从输入流中跳过所有前导的空白字符（空格、制表符、换行符）。
    // 这对于在读取混合数据时清除多余的空白很有用。
    cout << "Enter some text (with leading spaces if you like): ";
    string text_with_leading_spaces;
    cin >> ws >> text_with_leading_spaces; // 先跳过空白，再读取字符串
    cout << "You entered: '" << text_with_leading_spaces << "'" << endl;

    return 0;
}
```

int main() {

int main() {

## C++ 输入输出（I/O）优化

在 C++ 编程中，尤其是在需要处理大量数据输入的算法竞赛（如 ACM/ICPC）中，**标准输入输出（`cin` 和 `cout`）的速度可能会成为程序的瓶颈**，导致程序运行超时（**Time Limit Exceeded - TLE**）。为了解决这个问题，我们可以对 I/O 进行优化。

### 为什么需要优化 I/O？

`cin` 和 `cout` 是 C++ 标准库提供的、面向对象的 I/O 流。它们具有良好的类型安全性和易用性，但默认设置下，它们通常会比 C 语言风格的 `printf` 和 `scanf` 慢一些。主要原因有：

1.  **与 C 标准 I/O 的同步：** 默认情况下，C++ 流（`cin`/`cout`）会与 C 标准 I/O 流（`stdin`/`stdout`）保持同步。这意味着每次 C++ I/O 操作后，都会进行一次额外的检查，以确保两个系统的缓冲区状态一致，这会引入不必要的开销。
2.  **`cout` 绑定 `cin`：** 默认情况下，`cin` 被绑定到 `cout`。这意味着在每次 `cin` 进行输入操作之前，`cout` 会被强制刷新（flush）一次。这能确保你看到输出提示后再输入，但在大量输入时，频繁的刷新会降低效率。

-----

### I/O 优化方法

我们可以通过以下方法来优化 C++ 的 I/O 性能：

#### 1\. 关闭同步与解除绑定（最常用且有效）

这是最常用也最有效的优化手段。通常在 `main` 函数的开头添加以下两行代码：

```cpp
#include <iostream> // 使用 std::ios::sync_with_stdio 和 std::cin.tie 需要包含此头文件
using namespace std;
int main() {
    // 关闭 C++ I/O 流与 C 标准 I/O 流的同步
    // 一旦关闭，请不要混合使用 cin/cout 和 printf/scanf，否则可能导致输出顺序混乱或未定义行为。
    ios::sync_with_stdio(false);

    // 解除 cin 与 cout 的绑定
    // 这意味着 cin 在执行输入操作时不再强制刷新 cout。
    cin.tie(nullptr); // 或者 std::cin.tie(0); 效果相同

    // ... 你的程序代码 ...

    return 0;
}
```

**解释：**

  * `ios::sync_with_stdio(false);`：这条语句关闭了 `iostream` 与 `stdio` 之间的同步。这样，`cin` 和 `cout` 就可以独立地管理自己的缓冲区，大大提高了效率。
  * `cin.tie(nullptr);`：这条语句解除了 `cin` 和 `cout` 之间的绑定关系。通常，为了确保输出提示在输入之前显示，`cin` 会在每次读取前刷新 `cout`。解除绑定后，`cin` 不再等待 `cout` 刷新，从而提高了输入速度。

**重要提示：**
**一旦你使用了 `ios::sync_with_stdio(false);`，就绝对不要在同一个程序中同时使用 `cin`/`cout` 和 `scanf`/`printf`。** 否则，它们的缓冲区将不会同步，可能导致输入输出顺序错乱或数据丢失。你必须选择一种 I/O 方式并坚持使用。

#### 2\. 使用 `'\n'` 代替 `endl`

`endl` 会在输出换行符的同时强制刷新输出缓冲区，这会带来额外的性能开销。而 `'\n'` 仅仅是一个换行符，它不会强制刷新缓冲区。

```cpp
#include <iostream>

int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    // 推荐：使用 '\n' 换行，效率更高
    cout << "这是第一行。" << '\n';
    cout << "这是第二行。" << '\n';

    // 不推荐（除非确实需要立即刷新）：使用 std::endl 会强制刷新缓冲区
    // std::cout << "这一行会立即被刷新。" << std::endl;

    return 0;
}
```

**何时使用 `endl`：** 只有在需要立即将缓冲区内容输出到目标（例如，在调试时需要立即看到输出，或者在与用户进行交互式问答时需要确保提示信息立即显示）时，才应该使用 `endl`。

#### 3\. 考虑使用 `printf`/`scanf` 或自定义快读

如果关闭同步和解绑定后性能仍不满足要求，你可以考虑使用 C 风格的 `printf` 和 `scanf`。它们通常具有较低的开销。

```cpp
#include <cstdio> // 使用 printf 和 scanf 需要包含此头文件

int main() {
    int num;
    char str[100]; // char 数组用于 C 风格字符串

    // 使用 scanf 进行输入
    printf("Enter a number: ");
    scanf("%d", &num); // 注意：需要传入变量的地址

    printf("Enter a word: ");
    scanf("%s", str); // 注意：char 数组名本身就是地址，不需要 &

    // 使用 printf 进行输出
    printf("You entered: %d and %s\n", num, str);

    return 0;
}
```

对于极致的性能需求（例如在算法竞赛中，输入输出数据量非常庞大），一些选手还会实现自定义的**快读快写函数**。这些函数通常直接操作字符流，避免了标准库的抽象层开销，但实现起来更为复杂，且可读性较差。

-----

### 总结

在 C++ 中进行 I/O 优化时，最推荐的做法是：

1.  **在 `main` 函数开头添加 `ios::sync_with_stdio(false);` 和 `cin.tie(nullptr);`。**
2.  **始终使用 `'\n'` 替代 `endl` 来进行换行。**
3.  **坚持使用 `cin` 和 `cout`，并避免与 `scanf` / `printf` 混合使用。**

通过这些简单的优化，你的 C++ 程序在处理大量 I/O 时，性能通常会有显著提升。
