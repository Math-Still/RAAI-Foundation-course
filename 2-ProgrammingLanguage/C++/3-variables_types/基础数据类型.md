# 第三章节 基础数据类型

C++ 的类型系统由如下几部分组成：

1. 基础类型（括号内为代表关键词/代表类型）
   1. 无类型/`void` 型 (`void`)
   2. （C++11 起）空指针类型 (`std::nullptr_t`)
   3. 算术类型
      1. 整数类型 (`int`)
      2. 布尔类型/`bool` 型 (`bool`)
      3. 字符类型 (`char`)
      4. 浮点类型 (`float`,`double`)
2. 复合类型
   1. 引用类型
   2. 指针类型
   3. 数组类型
   4. 函数类型
   5. 枚举类型
   6. 类类型
      1. 非联合类型
      2. 联合类型

在这个章节中，我们只讲解基础类型

## 基本数据类型

C++提供了几种基本的数据类型：

| **类型大类** | **具体类型**    | **定义示例**                                         | **备注**                                                     |
| ------------ | --------------- | ---------------------------------------------------- | ------------------------------------------------------------ |
| **整数类型** | `int`           | `int age = 25;`                                      | 存储普通整数（如年龄、数量）                                 |
|              | `short`         | `short class_size = 50;`                             | 存储较小整数，节省内存（如分数）                             |
|              | `long`          | `long population = 1400000000L;`                     | 存储较大整数（如 ID、文件大小）                              |
|              | `long long`     | `long long distance = 9223372036854775807LL;`        | 存储极大整数（如天文数据）                                   |
| **浮点类型** | `float`         | `float pi = 3.14f;`                                  | 低精度小数（如游戏坐标）                                     |
|              | `double`        | `double precise_pi = 3.14159265359;`                 | 高精度小数（如科学计算、财务数据）                           |
|              | `long double`   | `long double ultra_precise = 3.141592653589793238L;` | 特殊工程计算                                                 |
| **字符类型** | `char`          | `char gender = 'M';`                                 | 单个字符（字母、符号），char类型变量在计算机中是以ASCII码值存储的。 |
|              | `unsigned char` | `unsigned char brightness = 255;`                    | 二进制数据（如图像像素）                                     |
| **布尔类型** | `bool`          | `bool is_student = true;`                            | 逻辑判断（开关状态、条件结果）                               |

## 变量声明和初始化

​在使用变量之前，所有变量必须在使用前进行声明，变量可以在使用前进行声明，不必在函数或过程的开始位置先声明。

比如我们定义一个整数类型的变量，然后对他进行赋值。

```cpp
#include<iostream> 
using namespace std;
int main() {
    // type var_name;
    int a;
    a = 1;
    cout<< a << endl;

}
```

这样就会输出变量a的值`1`

当然

变量名规范:

- 名称只能使用字母、数字、下划线。
- 名称的第一个字符不能是数字。
- 不能与C++关键字重名。

## 类型大小和范围

不同的数据类型占用不同的内存空间：

```cpp
#include <iostream>
#include <climits>

int main() {
    std::cout << "数据类型大小：" << std::endl;
    std::cout << "char: " << sizeof(char) << " bytes" << std::endl;
    std::cout << "int: " << sizeof(int) << " bytes" << std::endl;
    std::cout << "float: " << sizeof(float) << " bytes" << std::endl;
    std::cout << "double: " << sizeof(double) << " bytes" << std::endl;
    
    std::cout << "\n整数范围：" << std::endl;
    std::cout << "INT_MIN: " << INT_MIN << std::endl;
    std::cout << "INT_MAX: " << INT_MAX << std::endl;
    
    return 0;
}
```

当你运行上述程序，你能看到:

```sh
数据类型大小：
char: 1 bytes
int: 4 bytes
float: 4 bytes
double: 8 bytes

整数范围：
INT_MIN: -2147483648
INT_MAX: 2147483647
```

这里有是整数类型的全部取值范围

| 类型                  | 位数 | 最小值                     | 最大值                     |
| --------------------- | ---- | -------------------------- | -------------------------- |
| `signed char`         | 8    | -128                       | 127                        |
| `unsigned char`       | 8    | 0                          | 255                        |
| `short` / `short int` | 16   | -32,768                    | 32,767                     |
| `unsigned short`      | 16   | 0                          | 65,535                     |
| `int`                 | 32   | -2,147,483,648             | 2,147,483,647              |
| `unsigned int`        | 32   | 0                          | 4,294,967,295              |
| `long`                | 64   | -9,223,372,036,854,775,808 | 9,223,372,036,854,775,807  |
| `unsigned long`       | 64   | 0                          | 18,446,744,073,709,551,615 |
| `long long`           | 64   | -9,223,372,036,854,775,808 | 9,223,372,036,854,775,807  |
| `unsigned long long`  | 64   | 0                          | 18,446,744,073,709,551,615 |

> [!NOTE]
>
> 32 位系统差异：
>
> `long` 和 `unsigned long` 在 32 位系统中为 32 位，取值范围同 `int` 和 `unsigned int`
> 其他类型（如 `int`、`short` 等）位数和取值范围在 32 位与 64 位系统中保持一致

>下面我将以`int`为案例来讲解存储原理。
>
>首先，我们刚刚接触了两个概念：`byte`（字节） 和 `bit`（位），其中，位是计算机中最小的信息单位，只有两种状态 0 或 1 ，这也就是我们常说的二进制；字节是计算机中基本的存储单位，**1 字节 = 8 位**（即 `1 byte = 8 bits`）。同时，我们还需要了解原码，反码，补码。
>
>**原码**：最高位为符号位（正数为0，负数为1）再加上数字位，比如：在32位系统中，32 的原码为 `0000 0000 0000 0000 0000 0000 0010 0000` ，如果是-32，那么其原码为 `1000 0000 0000 0000 0000 0000 0010 0000` ，但这种方式有一个致命缺陷：**加减法运算需要额外处理符号位**，导致硬件设计复杂：
>
>- **加法**：若两数符号相同，直接加数值位；若符号不同，需先比较绝对值大小，用大数减小数，再取符号（类似人类手动计算）。
>- **减法**：需先将减法转为 “加上负数”，再重复上述加法的复杂逻辑。
>- **0的歧义**：`0000` 和 `1000` 都表示0。
>
>**反码**：正数的反码是其本身，负数的反码是在原码的基础上，符号位不变，其余各个位取反（即`2ⁿ-1-b`，b为原数字的绝对值）。比如：在32位系统中，32 的反码为 `0000 0000 0000 0000 0000 0000 0010 0000` ，如果是-32，那么其反码为 `1111 1111 1111 1111 1111 1111 1101 1111`。
>
>**补码**：正数的补码就是其本身，负数的补码是在原码的基础上，符号位不变，其余各位取反，最后+1.（也就是在反码的基础上+1，即`2ⁿ-b`）。比如：在32位系统中，32 的补码为 `0000 0000 0000 0000 0000 0000 0010 0000` ，如果是-32，那么其补码为 `1111 1111 1111 1111 1111 1111 1110 0000`
>
>为什么计算机要采用补码进行存储？
>
>1. **统一加减法**：计算`a-b`等价于计算`a+(-b)`，因为在模 `2ⁿ`的系统中，所以`2ⁿ`会自动舍去，因此可以直接用补码统一加减法
>2. **消除 “-0” 歧义**：补码中`0`只有一种表示（`0000...0000`），而原码中`-0`的位置被用来表示最小负数（如 32 位中`1000...0000`表示`-2^31`），扩大了表示范围。

## 变量作用域

作用域是变量可以发挥作用的代码块。

全局变量的作用域，自其定义之处开始，至文件结束位置为止。

局部变量的作用域，自其定义之处开始，至代码块结束位置为止。

由一对大括号括起来的若干语句构成一个代码块。

```c++
int g = 20;  // 定义全局变量 
int main() 
{  
 int g = 10;         // 定义局部变量  
 printf("%d\n", g);  // 输出 g  
 return 0; 
} 
```

如果一个代码块的内嵌块中定义了相同变量名的变量，则内层块中将无法访问外层块中相同变量名的变量。

例如上面的代码中，输出的值将是10。因此为了防止出现意料之外的错误，请尽量避免局部变量与全局变量重名的情况。

## 常量

常量是固定值，在程序执行期间不会改变，C++ 中有两种主要方式定义常量。

1. 使用 `const` 关键字。

   ```
   const int a = 2; a = 3; 
   ```

   如果修改了常量的值，在编译环节就会报错：`error: assignment of read-only variable‘a’`。

2. 用预处理指令`#define`

   ```c++
   #define PI 3.14159  // 定义常量PI
   // PI = 3.14;      // 错误：无法修改，因为#define是预处理替换
   
   int main() {
       double radius = 5.0;
       double area = radius * radius * PI;  
       return 0;
   }
   ```

   如果修改了常量的值，在编译环节会出现`error: expected unqualified-id before numeric constant`。

| **特性**     | **`const` 关键字**                           | **`#define` 预处理指令**                               |
| ------------ | -------------------------------------------- | ------------------------------------------------------ |
| **本质**     | 声明一个**只读变量**，具有类型检查           | 编译前的**文本替换**（无类型信息）                     |
| **作用域**   | 遵循变量作用域规则（如局部、全局、类作用域） | 从定义处到文件结束（或被`#undef`终止）                 |
| **内存占用** | 存储在常量存储区（占用内存）                 | 不占用内存（仅文本替换）                               |
| **类型安全** | ✅ 是（编译器进行类型检查）                   | ❌ 否（仅文本替换，无类型检查）                         |                        |                  |
| **替换时机** | 编译时使用变量值                             | 预处理时完成文本替换                                   |
| **代码示例** | `const int SIZE = 10;`                       | `#define SIZE 10`                                      |

## 如何使它崩溃

尝试以下操作来观察程序的行为：

1. **整数溢出**：

```cpp
int main() {
    int max_int = INT_MAX;
    std::cout << "Max int: " << max_int << std::endl;
    max_int = max_int + 1;  // 溢出
    std::cout << "After overflow: " << max_int << std::endl;
    return 0;
}
```

2. **未初始化变量**：

```cpp
int main() {
    int uninitialized;
    std::cout << "Uninitialized: " << uninitialized << std::endl;
    return 0;
}
```

3. **类型转换问题**：

```cpp
int main() {
    int x = 5;
    int y = 2;
    double result = x / y;  // 整数除法
    std::cout << "Integer division: " << result << std::endl;
    
    double result2 = static_cast<double>(x) / y;  // 浮点除法
    std::cout << "Float division: " << result2 << std::endl;
    return 0;
}
```
